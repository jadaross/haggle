# Haggle

Auto-messages Vinted buyers who favourite your listings.

When someone hearts one of your items, Haggle waits a natural-feeling 5–30 minutes, then drops them a personalised opening message generated by Claude — something that feels like a real seller saying hi, not a bulk DM. The goal is more conversations and more sales without sitting in the app.

## How it works

```
Vinted tab (browser)
  └─ extension service-worker polls /notifications every 10 min
       └─ new favourite detected → chrome.alarms schedules a send (5–30 min delay)
            └─ alarm fires → POST /events/favourite to backend
                 └─ backend: auth → deduplicate → rate-limit → Claude → persist
                      └─ returns message_text
                           └─ extension calls Vinted /conversations + /replies
                                └─ POST /events/{id}/sent to confirm
```

Two pieces:

- **Chrome extension** (`extension/`) — runs alongside an open Vinted tab, polls notifications, schedules sends, and uses the live browser session to deliver messages through Vinted's own conversation API. No Bearer tokens, no scraped credentials — it rides your real session.
- **FastAPI backend** (`backend/`) — validates the API key, deduplicates events, enforces rate limits, asks Claude for a message, and persists everything to Postgres. Deployed to Render.

## Modes

- **Auto** — sends as soon as the alarm fires.
- **Review** — drops the proposed message into a sidebar queue so you can approve, edit, or skip before it goes out.

Both modes respect a daily cap (per-key + a hard global ceiling) and a price floor so you don't accidentally negotiate against yourself.

## Backend

All commands run from `backend/`.

```bash
# Install (editable + dev extras)
pip install -e ".[dev]"

# Run locally (reads backend/.env)
uvicorn app.main:app --reload

# Tests
pytest

# Migrations
alembic revision --autogenerate -m "description"
alembic upgrade head
```

Copy `backend/.env.example` to `backend/.env` and fill in `ANTHROPIC_API_KEY`, `DATABASE_URL`, and `HAGGLE_SECRET_KEY`.

## Extension

No build step. Load it in Chrome:

1. Open `chrome://extensions`
2. Enable Developer mode
3. **Load unpacked** → select the `extension/` directory
4. Open the popup, paste your API key, pick a mode, and keep a `vinted.co.uk` tab open

## Deployment

Backend deploys to Render via `backend/render.yaml`. Pushing to `main` triggers a redeploy. `DATABASE_URL` is wired up automatically from the linked Render Postgres instance; `ANTHROPIC_API_KEY` is set manually in the Render dashboard.
